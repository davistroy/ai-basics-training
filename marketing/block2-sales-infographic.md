### Typography

### Iconography Style
- Workflow and automation-themed icons
- Connected nodes, gears, integration symbols
- Consistent with Block 1 style but more "engineered" feel
- Avoid: Code syntax, terminal windows, developer tools

### Layout Principles
- **F-pattern reading flow** â€” key information in top-left and down left side
- **Visual hierarchy** â€” largest elements are transformation and ROI sections
- **White space** â€” generous margins, not crowded
- **Continuity with Block 1** â€” recognizable as part of the same program

## COPY TONE GUIDELINES

### Voice Characteristics
- **Engineering-minded** but not developer-focused
- **Practical** with clear outcomes
- **Building** language (construct, automate, integrate)
- **Progress-focused** â€” you're advancing, not starting over

### Words/Phrases to USE
- "Automate" / "Workflow"
- "Integrate" / "Connect"
- "Scale" / "Multiply"
- "Quality gate" / "Evaluation"
- "Metrics" / "Performance"
- "MCP" / "Model Context Protocol"
- "Hands-off" / "Automatic"

### Words/Phrases to AVOID
- "Coding" / "Programming" / "Scripts"
- "API development" (MCP is configured, not coded)
- "Complex" (implies difficulty)
- "Technical debt" / "Refactoring"
- Any developer jargon beyond MCP
