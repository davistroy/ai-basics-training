## VISUAL DESIGN SPECIFICATIONS

### Color Palette
| Color | Usage | Hex Code |
|-------|-------|----------|
| Deep Navy | Headers, CTA background | #1a365d |
| Professional Blue | Accents, icons | #3182ce |
| Trust Teal | Success indicators | #319795 |
| Warm Gold | Certification, premium elements | #d69e2e |
| Clean White | Backgrounds, text contrast | #ffffff |
| Soft Gray | Secondary text, borders | #718096 |

### Typography
- **Headlines:** Bold sans-serif (Montserrat, Inter, or similar)
- **Body Text:** Clean, readable sans-serif (Open Sans, Source Sans)
- **Callouts:** Slightly larger, may use accent color

### Iconography Style
- Line-based or flat icons (not 3D or overly detailed)
- Consistent stroke width across all icons
- Business/professional aesthetic (no cartoon style)
- Avoid: Code brackets, terminal windows, programming symbols

### Layout Principles
- **F-pattern reading flow** â€” key information in top-left and down left side
- **Visual hierarchy** â€” largest elements are transformation and ROI sections
- **White space** â€” generous margins, not crowded
- **Mobile consideration** â€” design should work scaled down 50%

## COPY TONE GUIDELINES

### Voice Characteristics
- **Confident** but not arrogant
- **Practical** not theoretical
- **Direct** not corporate-speak
- **Peer-to-peer** not instructor-to-student
- **Results-focused** not feature-focused

### Words/Phrases to USE
- "Systematic" / "Structured"
- "Repeatable" / "Consistent"
- "Professional" / "Polished"
- "Practical" / "Hands-on"
- "Template" / "Framework"
- "Client-ready"
- "Time savings" / "Efficiency"

### Words/Phrases to AVOID
- "Coding" / "Programming" / "Development"
- "Technical" (without clarifying "no technical background needed")
- "Algorithm" / "Model" / "Neural network"
- "Easy" (implies it's trivial)
- "Revolutionary" / "Game-changing" (overused hype words)
- Any developer jargon (API, SDK, deployment, etc.)
