# Appendix: Participant Guide Template Guidelines

## Purpose of the Participant Guide

The participant guide serves as the primary self-study resource for each week. It should:

1. **Stand alone** - Participants can complete exercises without attending the live session
2. **Reinforce learning** - Recap key concepts from the workshop
3. **Guide practice** - Provide clear, step-by-step exercise instructions
4. **Enable self-assessment** - Help participants verify their own progress
5. **Connect forward** - Preview what's coming and why it matters

## Content Guidelines

### Key Concepts Section
- Define terms clearly and simply
- Include "Why It Matters" to motivate learning
- Provide concrete examples
- Note common mistakes to avoid
- Use visual aids where helpful

### Exercise Section
- Every step should be explicit - assume no prior knowledge
- Include file names and save locations
- Provide complete templates participants can copy
- Show at least one worked example
- Include troubleshooting for common issues

### Self-Assessment Section
- Align directly with learning objectives
- Make success criteria concrete and checkable
- Include reflection questions to deepen learning
- Provide clear exit criteria before proceeding

## Formatting Conventions

### Callouts
Use these markers for emphasis:

- **Tip:** Helpful advice
- **Warning:** Potential pitfall
- **Note:** Additional context
- **Important:** Critical information

### Code and Templates
- Use fenced code blocks for all code/templates
- Include language identifier for syntax highlighting
- Make templates copy-paste ready
- Add comments explaining key elements

### Checklists
- Use `- [ ]` for incomplete items
- Use `- [x]` for completed items in examples
- Keep checklists actionable and specific

## Quality Checklist

Before publishing each weekly participant guide:

- [ ] All learning objectives addressed
- [ ] Key concepts clearly explained with examples
- [ ] Exercise instructions are step-by-step
- [ ] Templates are complete and ready to use
- [ ] Examples show expected outputs
- [ ] Troubleshooting covers common issues
- [ ] Self-assessment aligns with objectives
- [ ] Links work and resources are accessible
- [ ] Navigation links are correct
- [ ] Glossary includes new terms
- [ ] Version history updated
