## Self-Evaluation Process (30 minutes)

### Block 1 Capstone Rubric

**Total Points: 30 (6 criteria × 5 points each)**

---

#### Criterion 1: Template Quality & Documentation (5 points)

| Score | Description |
|-------|-------------|
| **5 - Excellent** | All templates fully documented with metadata, purpose, variables, examples, and quality rubric links. Examples show actual AI output. Documentation is clear enough for someone else to use without explanation. |
| **4 - Good** | Templates well-documented with minor gaps. Examples present. Most metadata complete. |
| **3 - Adequate** | Templates have basic documentation. Some examples missing. Metadata partially complete. |
| **2 - Poor** | Minimal documentation. Few examples. Inconsistent structure. |
| **1 - Missing** | Templates lack documentation. No examples. Unusable by others. |

---

#### Criterion 2: Variety & Coverage (5 points)

| Score | Description |
|-------|-------------|
| **5 - Excellent** | 7+ templates covering diverse use cases. Clear evidence of using different prompting techniques (role prompting, few-shot, etc.). Templates address different deliverable types. |
| **4 - Good** | 5-6 templates with good variety. Multiple prompting techniques demonstrated. |
| **3 - Adequate** | 5 templates with some variety. At least 2 prompting techniques used. |
| **2 - Poor** | Fewer than 5 templates OR all templates very similar. |
| **1 - Missing** | Missing templates or no variety. |

---

#### Criterion 3: `style.json` Sophistication (5 points)

| Score | Description |
|-------|-------------|
| **5 - Excellent** | Comprehensive `style.json` covering voice, writing preferences, formatting, and domain-specific rules. Evidence of testing and refinement. Usage documentation included. |
| **4 - Good** | Good `style.json` with most sections complete. Some testing evident. |
| **3 - Adequate** | Basic `style.json` present. Core preferences defined. |
| **2 - Poor** | Minimal `style.json`. Missing key sections. |
| **1 - Missing** | No `style.json` or non-functional. |

---

#### Criterion 4: Quality System Implementation (5 points)

| Score | Description |
|-------|-------------|
| **5 - Excellent** | 2+ quality rubrics with specific, measurable criteria. Evidence of LLM-as-judge testing. Rubrics linked to templates. Comparison gates or other quality techniques demonstrated. |
| **4 - Good** | 2 quality rubrics with clear criteria. Some quality testing documented. |
| **3 - Adequate** | At least 1 quality rubric. Basic criteria defined. |
| **2 - Poor** | Minimal quality system. Vague criteria. |
| **1 - Missing** | No quality rubrics or evaluation system. |

---

#### Criterion 5: Repository Organization (5 points)

| Score | Description |
|-------|-------------|
| **5 - Excellent** | Clear taxonomy with consistent naming. Complete template index. README comprehensive. All folders documented. Easy to navigate and find templates. |
| **4 - Good** | Good organization. Template index present. README helpful. Minor inconsistencies. |
| **3 - Adequate** | Basic organization. Some structure. README exists but incomplete. |
| **2 - Poor** | Poor organization. Hard to find templates. Minimal documentation. |
| **1 - Missing** | No organization. No README. Chaotic structure. |

---

#### Criterion 6: Practical Application (5 points)

| Score | Description |
|-------|-------------|
| **5 - Excellent** | Task priority matrix complete with 10+ tasks analyzed. Clear evidence of using templates for real work. Specific examples of time saved or quality improved. Block 2 candidates identified. |
| **4 - Good** | Task matrix with good analysis. Evidence of practical use. Some impact documented. |
| **3 - Adequate** | Basic task matrix. Templates used for real work. Limited impact documentation. |
| **2 - Poor** | Minimal practical application. Theoretical rather than applied. |
| **1 - Missing** | No evidence of practical application. |
